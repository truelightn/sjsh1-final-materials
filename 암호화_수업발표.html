🔑 대칭키 방식 (비밀키)

암호화 키 = 복호화 키

🔐 비대칭키 방식 (공개키)

암호화 키 ≠ 복호화 키
자물쇠(공개키)는 누구나 가져가도 되고, 열쇠(개인키)는 나만 가진다.
느리지만 키를 따로 공유하지 않아도 안전하게 통신 가능.
예) RSA · ECC

단방향 암호화 기법

SHA — Secure Hash Algorithm

미국 국가안보국(NSA)이 설계하고 NIST가 표준으로 채택한 해시 함수 모음
입력 데이터가 달라지면 출력 해시값도 완전히 달라지는 눈사태 효과(Avalanche Effect)
SHA-1 (1995) — 사용 중단

출력 160비트
2017년 구글이 충돌 공격(SHAttered) 성공 → 현재 보안 용도 사용 불가

SHA-256 (2001) — 현재 표준 ★

출력 256비트 / 64자리 16진수
비밀번호 저장, TLS 인증서, 블록체인(Bitcoin) 등
충돌 발견 사례 없음 — 가장 널리 사용

SHA-384 / SHA-512

출력 384 / 512비트
더 높은 보안이 필요한 군사·금융 시스템에서 사용

SHA-3 (2015) — 차세대

SHA-2와 완전히 다른 구조(Keccak 알고리즘)
SHA-2가 뚫릴 경우를 대비한 백업 표준

💡 눈사태 효과 예시 — 입력 한 글자만 바꿔도 해시값이 완전히 달라진다!

레인보우 테이블 공격

단방향 암호화 기법

솔트 (Salt) — 레인보우 테이블 무력화

비밀번호에 무작위 문자열(Salt)을 붙인 뒤 해싱 — 같은 비밀번호도 전혀 다른 해시값이 나온다
레인보우 테이블은 Salt가 없는 값으로 만들어져 있어 즉시 무력화

RSA의 보안 원리

왜 RSA는 안전한가? — 소인수분해의 어려움

소인수분해(素因數分解)란?

어떤 수를 소수(Prime)들의 곱으로 나타내는 것
예) 30 = 2 × 3 × 5   /   15 = 3 × 5

서로소(互素)란?

두 수의 최대공약수(GCD) = 1인 관계
gcd(3, 40) = 1 ✓ 서로소  /  gcd(6, 40) = 2 ✗ 아님

✏️ 직접 구해보기 — 두 소수의 곱을 찾아라!

암호화의 이해
  • 🔑 데이터를 허가받지 않은 사람이 볼 수 없도록 변환
  • 📌 구성: 평문 · 암호문 · 키 · 알고리즘
  • 🔄 암호화(→) / 복호화(←) 과정
대표 알고리즘
  • 🔒 DES — 64bit · Feistel · 16라운드
  • 🔒 AES — 128bit · 4×4 행렬 · 10라운드
  • 🔓 RSA — 소인수분해 기반 공개키
  • 🔓 ECC — 타원 곡선 기반 효율적 공개키
  • 🧮 해시 — MD5, SHA (단방향)
💡 암호화는 디지털 시대에서 개인 정보와 데이터를 지키는 핵심 기술입니다!

예) 0x53 → 행 5, 열 3 → 0xED

⭐ "20이 왜 사라졌나?"
MixColumns는 행이 아닌 열(세로줄) 단위로 동작합니다.
열 0 = [20, 9D, 43, F0] → 4개의 값이 서로 뒤섞여 → [4F, 34, 30, 45]
20은 사라진 게 아니라, 다른 3개 값과 XOR·곱셈으로 섞여 새로운 숫자가 된 것입니다!
